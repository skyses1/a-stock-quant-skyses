# v0.5 修复版回测 — 修复清单与验证结果

## 修复的 5 个 P0 问题

### 1. 指数与个股数据无交集
**根因**: 之前用新浪获取指数(2025-07~2026-05) + 腾讯获取个股(2025-01~2025-05)，两者完全不重叠。
**修复**: 全部改用新浪 RAW 数据源，指数和个股使用同一 API，确保共同日期区间。
**结果**: 共同区间 310 天 (2025-02-18 ~ 2026-05-28)

### 2. 腾讯个股 K 线只到 2025-05
**根因**: `ingest_real_data.py` 的 `end_date` 参数写死为 `2025-06-01`。
**额外发现**: 腾讯接口返回数据为**倒序**（新→旧），原始代码取了前97条=最老的97天。
**修复**: 改用新浪 API，无日期限制，`datalen=300` 获取近300条。

### 3. 前复权价格扭曲收益率
**根因**: 腾讯 `qfqday` 返回前复权价格，复权因子调整导致单日涨幅可达 22%（A 股涨跌停限制 10%）。
**修复**: 
- 数据表增加 `price_type` 字段 (RAW/qfq)
- 回测强制 `WHERE price_type='raw'`
- 新浪 API 天然返回 RAW 价格

### 4. 最大回撤计算 Bug
**根因**: 原代码报告中 0.23%，实际应为 8.98%。Bug 在于 peak 初始化或计算逻辑有误。
**修复**: 使用标准 cummax 方法：
```python
peak = values[0]
for j in range(len(values)):
    if values[j] > peak: peak = values[j]
    dd = (peak - values[j]) / peak
    if dd > max_dd: max_dd = dd
```
**结果**: 修复后回撤 21.54%，峰值 114.7万(2025-09-08) → 谷值 90万(2026-02-02)

### 5. 买入执行价使用 close 而非 open
**根因**: T 日 08:00 的预测，买入价格应该用 T 日 open + 滑点。
**修复**: 
- 选股: 严格 `hist = [k for k in klines if k['date'] < cutoff_date]`
- 买入: `buy_price = bar['open'] * (1 + SLIPPAGE)`
- 市值: 仍用当日 close 计算（盘后估值）
